Data Cleaning en SQL

La limpieza de datos (SQL, Python, R) es una de las partes más importantes de un proyecto de análisis de datos, debido a que la calidad de los datos que utilizamos determina la precisión de los resultados y las decisiones que tomamos a partir de ellos.

Objetivo principal

Repasar las técnicas data cleaning más frecuentes en un set de datos.

Objetivos secundarios

  • Explorar el dataset en busca de incongruencias o errores.
  • Estandarizar el formato de fechas.
  • Dividir la información de la columna Address (street, state, zipcode).
  • Buscar y eliminar los valores duplicados en el dataset.
  • Eliminar las columnas que no se ocuparan en el analisis.

Acerca del dataset

El set de datos contiene información de las ventas de una tienda en línea durante 13 meses. El conjunto de datos consta de 10 columnas y 185,970 filas, cada una de las cuales representa un atributo de compra sobre un producto adquirido:

  • ID del pedido - Un ID único para cada pedido realizado sobre un producto.
  • Producto - Artículo que se compra.
  • Cantidad de productos - Describe cuántos productos ordenaron.
  • Precio unitario del producto - Precio de una unidad de ese producto.
  • Fecha del pedido - Fecha en la que se realiza el pedido.
  • Dirección de compra - Dirección a la que se envía el pedido.
  • Mes, Ventas, Ciudad, Hora - Atributos extra formados a partir de los anteriores columnas.

Fuente del dataset: Kaggle

Estandarizar el formato de fechas

El formato de la columna OrderDate incluye fecha y hora, es decir es una función 'DateTime'. Por tanto, se ha creado una nueva columna denominada order_date con el formato de fecha 'Date' para facilitar el posterior uso de las herramientas Business Intelligence.


Dividir la información de la columna Address

Normalmente, necesitamos analizar los datos por zona, región, ciudad, país, etc. Por eso es más útil tener la dirección de la casa, la ciudad y el estado en diferentes columnas.
En esta sección se hizo uso de las funciones "sbstr" e "instr" para extraer la información separada por un delimitador de texto de la columna PurchaseAddress. SQLite no cuenta con la función de CHARINDEX.


Buscar y eliminar los valores duplicados en el dataset

La función ROW_NUMBER asigna un número a cada fila con información que debe ser única en cada propiedad como orderID, address, product y muestra el número de veces que una fila con los mismos datos aparece en el conjunto de datos.


Para ello primero utilizamos la función row number para identificar los OrderID que tienen la información duplicada y filtramos aquellos en donde row number = 2 para determinar los 269 registros duplicados que tiene el dataset. Posteriormente, creamos una tabla temporal con la consulta anterior eliminando los registros duplicados.
